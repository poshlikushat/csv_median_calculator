# csv_median_calculator

Консольное приложение на C++23 для расчёта инкрементальной медианы цен из CSV-файлов с потоковыми данными биржевых торгов.

По мере обработки каждой новой цены программа пересчитывает медиану с учётом всех ранее считанных значений. Когда медиана изменяется — фиксирует это в результирующем файле вместе с временной меткой. Несколько входных файлов загружаются параллельно через `std::jthread`, после чего записи объединяются и сортируются по `receive_ts`.

## Сборка проекта

```bash
# Клонировать репозиторий
git clone <url>
cd csv_median_calculator

# Сконфигурировать и собрать (Release)
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
```

## Формат конфигурации

Конфигурация задаётся в файле TOML:

```toml
[main]
# Путь к директории с входными CSV-файлами (обязательный)
input = "examples/input"

# Путь к директории для выходного файла (опциональный)
# По умолчанию создаётся директория "output" рядом с исполняемым файлом
output = "examples/output"

# Маски имён файлов для фильтрации (опциональный)
# Файл включается если его имя содержит хотя бы одну маску
# Пустой список [] — читаются все CSV-файлы в директории
filename_mask = ["level", "trade"]
```

**Параметры:**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `main.input` | string | ✅ | Путь к директории с входными CSV |
| `main.output` | string | ❌ | Путь к директории для результата (по умолчанию `./output`) |
| `main.filename_mask` | array of strings | ❌ | Маски фильтрации файлов (по умолчанию все CSV) |

Выходной файл всегда называется `median_result.csv` и создаётся внутри указанной `output`-директории. Директория создаётся автоматически, если не существует.

## Формат входных данных

### `level.csv` — данные стакана

```csv
receive_ts;exchange_ts;price;quantity;side;rebuild
1716810808593627;1716810808574000;68480.00000000;10.10900000;bid;1
1716810808593627;1716810808574000;68479.90000000;0.00400000;bid;0
```

| Поле | Тип | Описание |
|---|---|---|
| `receive_ts` | int64 | Временная метка получения (микросекунды) |
| `exchange_ts` | int64 | Временная метка биржи (микросекунды) |
| `price` | double | Цена |
| `quantity` | double | Объём |
| `side` | string | Сторона: `bid` или `ask` |
| `rebuild` | int | Флаг перестроения стакана: 0 или 1 |

### `trade.csv` — данные сделок

```csv
receive_ts;exchange_ts;price;quantity;side
1716810808663260;1716810808661000;68480.10000000;0.01100000;bid
1716810809314641;1716810809312000;68480.00000000;0.00200000;ask
```

| Поле | Тип | Описание |
|---|---|---|
| `receive_ts` | int64 | Временная метка получения (микросекунды) |
| `exchange_ts` | int64 | Временная метка биржи (микросекунды) |
| `price` | double | Цена сделки |
| `quantity` | double | Объём сделки |
| `side` | string | Сторона: `bid` или `ask` |

## Формат выходных данных

Файл `median_result.csv` — строка записывается **только при изменении медианы**:

```csv
receive_ts;price_median
1716810808593627;0.00000000
1716810808593629;68480.20000000
1716810809314641;68479.97000000
1716810809719209;68480.15885417
1716810810150000;68480.03134593
```

| Поле | Описание |
|---|---|
| `receive_ts` | Временная метка события, изменившего медиану |
| `price_median` | Значение медианы на основе всех цен до этого момента |

> **Примечание по точности:** используется приближённый алгоритм p² из `Boost.Accumulators`. На малых выборках (< 100 элементов) возможна погрешность; на больших выборках алгоритм сходится к точному значению.

## Примеры использования

### Запуск на примерах из репозитория

```bash
./build/csv_median_calculator --config config.toml
```

Входные файлы берутся из `examples/input/`, результат сохраняется в `examples/output/median_result.csv`.

### Обработка только сделок

```toml
[main]
input = "./data"
output = "./results"
filename_mask = ["trade"]
```

### Обработка всех CSV без фильтрации

```toml
[main]
input = "./data"
filename_mask = []
```
